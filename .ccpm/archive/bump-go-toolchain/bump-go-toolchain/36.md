# Task: Update CI to derive Go version from go.mod

## Description

Update `.github/workflows/release.yml` to read the Go version from `go.mod` instead of hardcoding `"1.25.0"`. Replace the hardcoded `go install wails@v2.11.0` with `go tool wails` which uses the version pinned in `go.mod`'s `tool` directive (v2.12.0). Also update `wails build` to use `go tool wails build` for consistency with the local Makefile.

## Acceptance Criteria
- [ ] `release.yml` uses `go-version-file: go.mod` instead of `go-version: "1.25.0"`
- [ ] The "Install Wails CLI" step is removed — replaced by `go tool wails` usage
- [ ] The "Build with Wails" step uses `go tool wails build` instead of bare `wails build`
- [ ] No hardcoded Go or Wails version strings remain in CI workflow files

## Technical Details

1. In the "Set up Go" step, replace:
   ```yaml
   go-version: "1.25.0"
   ```
   with:
   ```yaml
   go-version-file: go.mod
   ```
   `actions/setup-go` v4+ parses the `go` directive from `go.mod` to determine which version to install.

2. Remove the "Install Wails CLI" step entirely:
   ```yaml
   - name: Install Wails CLI
     run: go install github.com/wailsapp/wails/v2/cmd/wails@v2.11.0
   ```
   The `tool` directive in `go.mod` already pins wails v2.12.0. `go tool wails` will auto-compile it on first use.

3. In the "Build with Wails" step, replace `wails build` with `go tool wails build` so it uses the project-pinned version.
