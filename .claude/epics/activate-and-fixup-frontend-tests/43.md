# Task: Install Vitest dependencies and create config files

## Description
Install the Vitest test framework with supporting libraries and create the configuration files needed for unit and component testing. Follow the EisenKan reference at `tmp/eisenkan/frontend/vitest.config.ts` and `tmp/eisenkan/frontend/vitest.setup.ts`.

## Acceptance Criteria
- [ ] `vitest`, `@testing-library/svelte`, and `jsdom` added as devDependencies in `frontend/package.json`
- [ ] `"test": "vitest"` script added to `frontend/package.json`
- [ ] `frontend/vitest.config.ts` created with: jsdom environment, Svelte plugin with `configFile: false`, `resolve.conditions: ['browser']`, setup file reference, exclusion of `src/lib/wails/` from test discovery
- [ ] `frontend/vitest.setup.ts` created with HTMLDialogElement polyfill (from EisenKan reference)
- [ ] `npx vitest --run` executes successfully (even with 0 tests initially)
- [ ] Existing `npm run dev`, `npm run build`, `npm run check`, and `npm run lint` still work

## Technical Details
- Reference: `tmp/eisenkan/frontend/vitest.config.ts` and `tmp/eisenkan/frontend/vitest.setup.ts`
- Bearing does NOT use Tailwind CSS, so omit the `tailwindcss()` plugin from vitest.config.ts (unlike EisenKan)
- Use `configFile: false` on the Svelte plugin to disable vitePreprocess in tests
- Exclude `src/lib/wails/` via Vitest's `exclude` config to avoid test discovery in generated files
- Run `cd frontend && npm install` after modifying package.json
