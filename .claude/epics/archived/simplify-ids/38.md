# Task: Update mock data and mock bindings

## Description
Update all mock data and mock binding functions in `wails-mock.ts` to use flat IDs with `parentId` fields, ensuring the browser-based testing variant works correctly.

## Acceptance Criteria
- [ ] All mock theme/objective/KR IDs use flat format (`THEME-1`, `OBJ-1`, `KR-1`, `TASK-1`)
- [ ] All mock objectives have `parentId` set to their parent theme or objective ID
- [ ] All mock key results have `parentId` set to their parent objective ID
- [ ] `CreateObjective()` generates flat `OBJ-N` IDs and sets `parentId`
- [ ] `CreateKeyResult()` generates flat `KR-N` IDs and sets `parentId`
- [ ] `CreateTask()` generates `TASK-N` IDs
- [ ] Mock task IDs use `TASK-N` format
- [ ] Browser testing at localhost:5173 works correctly

**Files affected:**

- `THEME-01.OKR-01.KR-01` → `KR-1` with `parentId: 'OBJ-1'`
