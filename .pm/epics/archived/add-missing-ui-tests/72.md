## Technical Details

**Mock bindings needed:** `GetTasks`, `GetThemes`, `CreateTask`, `MoveTask`, `DeleteTask`, `GetTasksFiltered`, `LoadNavigationContext`, `SaveNavigationContext`

**Test cases (minimum 10):**
1. Renders three Kanban columns (todo, doing, done) with headers
2. Renders task cards in correct columns based on status
3. Task cards display theme color border, priority badge, and title
4. Opens create dialog when clicking "+ New Task" button
5. Create button is disabled when title is empty
6. Submitting create dialog calls CreateTask with correct args
7. Delete button calls DeleteTask and removes card optimistically
8. Priority sorting: Q1 tasks appear before Q2, Q2 before Q3 in todo column
9. Filters tasks by filterThemeId prop
10. Filters tasks by filterDate prop
11. Shows empty column placeholder when column has no tasks
12. Shows error banner on API failure, dismissible

**Key selectors:**
- `.eisenkan-container` — root container
- `.kanban-board` — board layout
- `.kanban-column` — column containers
- `.column-header`, `.task-count` — column headers
- `.task-card` — task cards
- `.priority-badge` — Q1/Q2/Q3 labels
- `.create-btn` — new task button
- `.delete-btn` — delete button on cards
- `.dialog-overlay`, `.dialog` — create modal
- `.error-banner` — error state
- `.loading-state` — loading state
- `.empty-column` — empty column placeholder
