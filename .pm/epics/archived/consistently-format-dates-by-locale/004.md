# Task: Migrate EisenKanView date displays

## Description
Replace all date display code in `EisenKanView.svelte` with calls to the shared `date-format.ts` utility, ensuring locale-consistent formatting for the header "today" display, task card dates, and context menu day labels.

## Acceptance Criteria
- [ ] Header "today" display uses `formatDateLong()` instead of `toLocaleDateString(undefined, ...)`
- [ ] Task card `{task.dayDate}` rendered via `formatDate()` instead of raw ISO string
- [ ] Context menu "Go to Day" label uses `formatDate()` instead of raw ISO string
- [ ] No direct `toLocaleDateString` calls remain in EisenKanView
- [ ] No raw ISO date strings displayed to users
- [ ] Existing EisenKanView tests pass (update assertions if needed)
- [ ] Both native and browser-based app display dates correctly

## Technical Details
- Import `formatDate`, `formatDateLong` from `../lib/utils/date-format`
- Replace `formatToday()` function body: use `formatDateLong(new Date().toISOString().split('T')[0])`
- Replace `{task.dayDate}` in task card template with `{formatDate(task.dayDate)}`
- Replace `{contextMenuTask.dayDate}` in context menu with `{formatDate(contextMenuTask.dayDate)}`
- Files affected: `frontend/src/views/EisenKanView.svelte`, `frontend/src/views/EisenKanView.test.ts`
