# Task: Create Dialog.svelte shared component

## Description
Create a reusable `Dialog.svelte` component in `frontend/src/lib/components/` that encapsulates the modal overlay + container pattern currently duplicated in CreateTaskDialog, EditTaskDialog, and CalendarView.

## Acceptance Criteria
- [ ] `Dialog.svelte` created in `frontend/src/lib/components/`
- [ ] Props: `title` (string), `maxWidth` (string, default `"400px"`)
- [ ] Renders overlay backdrop (semi-transparent black)
- [ ] Renders centered white container with title
- [ ] Provides snippet/slot for body content
- [ ] Provides snippet/slot for footer actions (button row)
- [ ] Clicking overlay triggers close callback
- [ ] Uses design tokens for colors, radius, spacing
- [ ] Unified z-index: 1000
- [ ] Exported from `frontend/src/lib/components/index.ts`
- [ ] `make frontend-lint` and `make frontend-check` pass

## Technical Details
- Reference existing patterns from:
  - `.dialog-overlay` + `.dialog` in CreateTaskDialog.svelte (canonical)
  - `.dialog-overlay` + `.dialog` in EditTaskDialog.svelte (identical)
  - `.dialog-overlay` + `.dialog` in CalendarView.svelte (z-index differs)
- Overlay: fixed position, full viewport, rgba(0,0,0,0.5), flex center
- Container: white bg, `var(--radius-lg)` border-radius, padding, max-height 90vh with overflow-y auto
- Title: h2, font-size 1.25rem, bold
- Actions slot: flex end, gap
