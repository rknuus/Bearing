# Task: Extend CreateTask backend API and Wails binding with new parameters

## Description
Add description, tags, dueDate, and promotionDate parameters to the `CreateTask` method in PlanningManager and the Wails App binding. These fields are already supported by the Task struct and SaveTask — they just aren't settable at creation time.

## Acceptance Criteria
- [ ] `PlanningManager.CreateTask` accepts description (string), tags (string, comma-separated), dueDate (string), promotionDate (string) in addition to existing params
- [ ] `App.CreateTask` in main.go passes through the new parameters
- [ ] Non-empty dueDate and promotionDate are validated as YYYY-MM-DD format
- [ ] Tags string is split into a []string slice on the Task struct
- [ ] Empty optional fields are handled gracefully (no validation errors)
- [ ] Existing Go tests updated for new signature
- [ ] New tests for the optional fields (create with description, create with tags, etc.)
- [ ] `make test` passes

## Technical Details
- Files affected:
  - `internal/managers/planning_manager.go` — extend `CreateTask` signature and populate new fields on the Task struct
  - `internal/managers/planning_manager_test.go` — update all `CreateTask` call sites with new params, add tests for optional fields
  - `main.go` — update `App.CreateTask` binding signature
- Consider whether Wails v2 handles 8 positional string params well, or if a struct param is needed. Check existing patterns in the codebase.
