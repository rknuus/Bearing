# Task: Create Button.svelte shared component

## Description
Create a reusable `Button.svelte` component in `frontend/src/lib/components/` that encapsulates the 4 button variants currently duplicated across the codebase: primary, secondary, icon, and danger.

## Acceptance Criteria
- [ ] `Button.svelte` created in `frontend/src/lib/components/`
- [ ] Supports `variant` prop: `"primary"` (default), `"secondary"`, `"icon"`, `"danger"`
- [ ] Supports `disabled`, `type` props
- [ ] Icon variant supports `color` prop for sub-variants (edit=blue, delete=red, add=green, etc.)
- [ ] Uses design tokens from Task 001 for all color/radius values
- [ ] Spreads rest props for `onclick`, `class`, etc.
- [ ] Slot for button content
- [ ] Exported from `frontend/src/lib/components/index.ts`
- [ ] `make frontend-lint` and `make frontend-check` pass

## Technical Details
- Reference existing styles from:
  - `.btn-primary` in CreateTaskDialog.svelte / EditTaskDialog.svelte (canonical primary)
  - `.btn-secondary` in CreateTaskDialog.svelte / EditTaskDialog.svelte (canonical secondary)
  - `.icon-button` in OKRView.svelte (canonical icon with color variants)
  - `.create-btn` in EisenKanView.svelte (same as primary)
- Normalize border-radius to `var(--radius-md)` (6px) for all non-icon variants
- Normalize border-radius to `var(--radius-sm)` (4px) for icon variant
- Primary: blue bg, white text, darker hover, disabled state
- Secondary: white bg, gray text, gray border, light hover
- Danger: red bg, white text, darker hover
- Icon: transparent bg, hover gray bg, color variants via prop
