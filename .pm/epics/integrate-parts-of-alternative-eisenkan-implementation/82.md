# Task: Create EditTaskDialog component

## Description

Create an EditTaskDialog Svelte component for modifying existing tasks. The dialog opens when clicking a task card in the EisenKan board. It provides fields for title, description, priority (with quadrant visualization), tags, due date, and promotion date. Save calls `UpdateTask` on the backend; cancel discards changes.

## Acceptance Criteria

- [ ] `EditTaskDialog.svelte` component created in `frontend/src/components/`
- [ ] Dialog opens when clicking a task card in EisenKanView
- [ ] Editable fields: title (text input), description (textarea), priority (quadrant selector), tags (comma-separated input or tag chips), due date (date input), promotion date (date input)
- [ ] Priority visualized as Eisenhower quadrant selector (Q1/Q2/Q3/Q4)
- [ ] Form pre-populated with current task values on open
- [ ] Save button calls `UpdateTask` API and closes dialog
- [ ] Priority changes cause task to re-sort in Todo column after save
- [ ] Cancel button / Escape key discards changes and closes dialog
- [ ] Save button disabled during submission (`isSubmitting` state)
- [ ] Error message displayed if save fails (rule violation feedback)
- [ ] Dialog uses modal overlay (consistent with existing create dialog styling)
- [ ] Works in both Wails native and browser mock variants
- [ ] Unit tests for dialog open/close, form population, save, cancel, error handling
