# Task: Create TaskFormFields shared component

## Description

Create a new `TaskFormFields.svelte` component that renders the 6 shared task form fields: title, theme, description, tags, due date, and promotion date. This component will be used by both CreateTaskDialog and EditTaskDialog.

## Acceptance Criteria

- [ ] New file `frontend/src/components/TaskFormFields.svelte` created
- [ ] Renders all 6 fields: title (text), theme (select), description (textarea), tags (text), due date (date), promotion date (date)
- [ ] All field values use `$bindable()` props so parents can two-way bind
- [ ] Accepts `themes: LifeTheme[]` prop for the theme selector
- [ ] Accepts `disabled: boolean` prop to disable all fields
- [ ] CSS for `.form-group`, `.form-row`, and input styles included in the component
- [ ] Labels and placeholders match current dialogs: "Title", "Theme", "Description", "Tags (comma-separated)", "Due Date", "Promotion Date"
- [ ] Due date and promotion date are in a side-by-side `.form-row` layout

## Technical Details

- Props interface:
  ```typescript
  interface Props {
    title: string;       // $bindable()
    themeId: string;     // $bindable()
    description: string; // $bindable()
    tags: string;        // $bindable() â€” comma-separated string
    dueDate: string;     // $bindable()
    promotionDate: string; // $bindable()
    themes: LifeTheme[];
    disabled?: boolean;
  }
  ```
- CSS extracted from CreateTaskDialog's `.form-group`, `.form-row`, and input focus styles
- Tags remain a comma-separated string in the UI; conversion to `string[]` is the parent's responsibility
