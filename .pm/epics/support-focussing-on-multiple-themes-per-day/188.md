# Task: Migration script — themeId to themeIds

## Description

Create a standalone Go migration script that converts existing calendar JSON files from `themeId: string` to `themeIds: []string`. This is a one-time utility, not part of production code.

## Acceptance Criteria

- [ ] Script reads all `~/.bearing/data/calendar/*.json` files
- [ ] Each DayFocus entry: `"themeId": "X"` → `"themeIds": ["X"]`, old `"themeId"` key removed
- [ ] Empty `"themeId": ""` → `themeIds` field omitted (or empty array)
- [ ] Idempotent: if `themeIds` already present and `themeId` absent, skip
- [ ] Script lives in `scripts/` directory (not imported by production code)
- [ ] Script commits the changes via go-git after successful migration
- [ ] Script can be run with `go run scripts/migrate-theme-id-to-theme-ids.go`

## Technical Details

### Files affected
- `scripts/migrate-theme-id-to-theme-ids.go` (new file)

### Approach
- Use `encoding/json` with `map[string]interface{}` to read/modify JSON without needing the DayFocus struct (avoids import dependency on internal packages)
- For each year JSON file:
  1. Read and unmarshal as `[]map[string]interface{}`
  2. For each entry: if `themeId` key exists, set `themeIds: [themeId]`, delete `themeId`
  3. Marshal back with indentation and write
- After all files processed, commit with go-git

### Edge cases
- File doesn't exist or directory is empty → no-op
- Entry has both `themeId` and `themeIds` → prefer `themeIds`, delete `themeId`
- Entry has no `themeId` key → skip
